Supabase RLS Blocking Data in Your AI-Built App — How to Diagnose It

If your app UI loads but data disappears, Supabase RLS may be blocking access. Do not change policies blindly.

Break type checklist

  • Data loads for one user but not another.
  • Tables show data in Supabase but the app returns empty arrays.
  • Login works, but dashboard data does not load.
  • Console or network tab shows permission denied.

What to check first

Confirm the authenticated user exists, the client uses the correct anon key, the query targets the correct table, and RLS policies allow the current user to read the rows.

Paste-ready prompt

Do not rewrite the app. My Supabase-backed AI-built app is not loading data. Diagnose whether this is RLS, auth context, wrong table/query, or environment configuration. Do not change database schema or policies until you explain the evidence. Give me the smallest safe check and fix.
