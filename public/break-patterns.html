VibeFix Break Pattern Database — Real AI App Breaks and Their Root Causes

Use these patterns to identify what likely broke before another broad AI prompt makes recovery harder.

App broke after update

Builder tool: Lovable, v0, Replit

Root cause: A recent generated edit changed more than the intended feature, usually touching shared state, routes, auth redirects, or layout wrappers.

What not to touch: Database schema, auth rules, production env vars.

Fix approach: Compare last working state, isolate changed files, fix forward only if the break is small.

Confidence: Medium when the last prompt is known.

UI/layout broke

Builder tool: Lovable, Bolt, v0

Root cause: Component hierarchy, CSS scope, responsive constraints, or shared layout classes changed unexpectedly.

What not to touch: Auth, database, payment, env settings.

Fix approach: Restore layout boundaries and test mobile/desktop before functional changes.

Confidence: High when screenshots show layout-only failure.

Login/signup broke

Builder tool: Lovable, Cursor, Replit

Root cause: Redirect URLs, session handling, auth provider config, or client/server boundary changed.

What not to touch: Production auth rules until logs identify the failed step.

Fix approach: Check callback URLs, console errors, provider config, and affected route.

Confidence: Medium without exact auth error.

Database/data not loading

Builder tool: Lovable, Bolt, Cursor

Root cause: Supabase RLS/Firebase rules, API query change, table mismatch, or missing env variable.

What not to touch: Table structure and policies unless the error proves access failure.

Fix approach: Inspect network response and permission errors before UI work.

Confidence: High with exact API/RLS error.

Preview works, production fails

Builder tool: Bolt, Replit, Lovable

Root cause: Environment variables, production URLs, auth redirect mismatch, or build-only behavior.

What not to touch: UI components until deploy/config is checked.

Fix approach: Compare preview vs production env/config and deploy logs.

Confidence: High when preview works consistently.

Deployment issue

Builder tool: Replit, Bolt, Cursor

Root cause: Build command, output directory, missing package, or runtime-specific API use.

What not to touch: Feature logic before the build error is resolved.

Fix approach: Read the first real build error and fix the smallest config/package issue.

Confidence: High with full build logs.

Payment/checkout issue

Builder tool: Any

Root cause: Wrong checkout link, inactive payment store, missing redirect, or frontend/backend mismatch.

What not to touch: Secret keys and webhook code unless you know the payment flow.

Fix approach: Verify live payment link and success/intake redirect first.

Confidence: Medium without provider logs.

One feature broke another

Builder tool: Cursor, Claude Code, Lovable

Root cause: Shared component, state, API handler, or utility changed for the new feature.

What not to touch: Unrelated working flows.

Fix approach: Open diff, protect old flow, apply smallest patch.

Confidence: Medium-high with changed file list.

AI keeps fixing one thing and breaking another

Builder tool: Cursor, Claude Code, Lovable

Root cause: Prompts are too broad and the AI is refactoring instead of isolating.

What not to touch: Do not allow full rewrites or broad cleanup prompts.

Fix approach: Use scoped prompts with explicit no-touch boundaries.

Confidence: High when multiple fix attempts caused regressions.

Environment variable/config issue

Builder tool: Bolt, Replit, Cursor

Root cause: Preview and production do not share the same API keys, URLs, or feature flags.

What not to touch: App logic until config parity is checked.

Fix approach: Compare env names, values, deploy target, and runtime logs.

Confidence: High when failure is production-only.
